# ADR-002: New Include Namespaces Match Bounded Context Directories

## Status

Accepted

## Date

2026-06-25

## Context

Moving source files to `libs/<context>/` requires deciding whether `#include` paths should reflect the new locations or remain unchanged.

Current paths:
- `#include <ymir/core/RigidBody6DOF.h>`
- `#include <ymir/naval/forces/InertialForces.h>`

After the move, the physical path of headers changes. The question is whether the include namespace changes too.

## Decision

Include namespaces change to match the new directory structure. Every `#include` path is updated atomically as part of the same commit that moves the files.

New namespaces:
- `ymir/physics/` — bodies, force model, integrator, all force implementations
- `ymir/world/` — wave engine, environment
- `ymir/vessel/` — vessel config, naval context
- `ymir/simulation/` — simulation orchestration
- `ymir/persistence/` — JSON I/O
- `ymir/common/` — math utilities, physical constants, shared types

## Alternatives Considered

### Alternative 1: Keep Existing Namespaces

- **Description**: Move files but keep `#include <ymir/core/...>` and `#include <ymir/naval/...>` by placing them under the old namespace subdirectories inside the new locations (e.g., `libs/physics/include/ymir/core/...`)
- **Pros**: Zero changes to existing includes; no risk of missed paths
- **Cons**: Directory named `libs/physics/` containing headers under `ymir/core/` — confusing and inconsistent; defeats the purpose of the rename; future developers would need to know the mapping
- **Why rejected**: Inconsistency between folder names and include namespaces creates confusion and forces every contributor to maintain a mental translation table

## Consequences

### Positive

- Include path is a self-documenting hint about which module a header belongs to
- Consistent: the folder name, the CMake target name, and the include prefix all say the same thing
- Eliminates the question "is this a core thing or a naval thing?"

### Negative

- Every existing `#include` in source files and tests must be updated — approximately 50 include directives across tests, source files, and headers
- One missed include causes a compile error (detectable immediately, not a silent regression)

### Risks

- **Missed includes**: Run `grep -r "ymir/core\|ymir/naval\|ymir/adapters"` after moving to verify no stale paths remain. CI catches any misses.
