# ADR-001: Phased Rollout — Uniform Conditions Before Grid Maps

## Status

Accepted

## Date

2026-06-28

## Context

The Environmental Conditions feature encompasses three sub-capabilities of increasing complexity: uniform conditions (direction + speed), grid-mapped spatial fields, and real-time instructor control during simulation. Delivering all three simultaneously increases scope and risk. The physics foundation (Environment C++ class, NavalDomain, force models) is already complete; the gap is in the scenario data model, WASM bindings, and UI.

## Decision

Deliver the feature in three phases:
- **Phase 1**: Uniform conditions with temporal keyframe evolution for all three types (current, wind, wave). Multiple simultaneous conditions that compose vectorially. Configured before simulation starts.
- **Phase 2**: Grid-mapped spatial fields (current and wave maps) with the same keyframe and composition model.
- **Phase 3**: Real-time instructor control — the ability to add or modify conditions while a simulation is running.

## Alternatives Considered

### Alternative 1: Full Delivery in One Phase

- **Description**: Implement uniform, grid maps, keyframes, and composition all at once.
- **Pros**: Single release; no phased dependencies.
- **Cons**: Grid maps introduce significant UI complexity (file import, map visualization, cell-level interpolation). Blocking Phase 1 value on the hardest part.
- **Why rejected**: Risk of delaying uniform conditions — which provide immediate simulation realism — behind the more complex grid feature.

### Alternative 2: Grid-Only (No Uniform)

- **Description**: Skip uniform conditions and go directly to grid maps.
- **Pros**: More powerful from the start.
- **Cons**: Grid maps require external data files; most training scenarios only need simple uniform conditions. No fallback for simple setups.
- **Why rejected**: Overcomplicated MVP; uniform conditions cover the majority of real use cases.

## Consequences

### Positive

- Phase 1 delivers simulation realism (current drag, wind force, wave excitation) with minimal scope.
- The entire WASM binding → worker → store → UI pipeline is validated with simple data before grid complexity is added.
- Each phase is independently useful and shippable.

### Negative

- Instructors who need grid-mapped currents (port approach scenarios with real bathymetric data) must wait for Phase 2.

### Risks

- Phase 2 grid import UX must be designed to be consistent with Phase 1 uniform UX — risk of divergent patterns if not planned together.
