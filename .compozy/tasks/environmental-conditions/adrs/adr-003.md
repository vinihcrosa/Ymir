# ADR-003: Independent Evaluation + Vector Sum for Multiple Grid Maps

## Status

Accepted

## Date

2026-06-28

## Context

When an instructor adds multiple grid-mapped conditions of the same type (e.g., two current maps covering different areas, or one tidal current and one river discharge map), the system needs a strategy to compute the resultant at any vessel position. The two main options are: reproject both grids onto a unified grid, or evaluate each grid independently at the vessel's position and sum the resulting vectors.

## Decision

Each grid is evaluated independently at the vessel's position using bilinear interpolation within its own coordinate system. The resulting vectors are summed. Grids of different size, resolution, or origin can coexist without any reprojection or resampling step.

Behavior outside a grid's covered area: velocity = zero (no nearest-neighbor extrapolation). This diverges from the MATLAB `dynamics` reference which uses `'nearest'` extrapolation.

## Alternatives Considered

### Alternative 1: Unified Grid Reprojection

- **Description**: Before simulation starts, all grid maps of the same type are reprojected onto a single unified grid by superimposing their values cell by cell.
- **Pros**: Single lookup per step at runtime; no per-grid interpolation overhead.
- **Cons**: Requires defining a target resolution and extent upfront. Reprojection step adds preprocessing complexity and may introduce interpolation artifacts at mismatched resolutions.
- **Why rejected**: Adds preprocessing complexity with no meaningful benefit for the vessel-count and grid sizes expected in this domain.

### Alternative 2: Nearest-Neighbor Outside Grid (match dynamics)

- **Description**: Use `'nearest'` extrapolation — when the vessel is outside the grid, return the value of the nearest edge cell (matching MATLAB dynamics behavior).
- **Pros**: Continuous force field; no sudden force discontinuity when vessel exits grid.
- **Cons**: Unintuitive — edge currents applied far outside their intended area.
- **Why rejected**: Zero-outside is more predictable and easier for instructors to reason about. The divergence from dynamics is intentional.

## Consequences

### Positive

- Supports overlapping grids of arbitrary resolution without preprocessing.
- Simple, predictable rule for out-of-area behavior.

### Negative

- Force discontinuity at grid boundary (step from nonzero to zero). May produce visible artifacts at grid edges in some scenarios.

### Risks

- If a grid boundary happens to be at a critical maneuvering point, the discontinuity could affect simulation realism. Mitigation: instructors should size their grids to extend well beyond the area of interest.
