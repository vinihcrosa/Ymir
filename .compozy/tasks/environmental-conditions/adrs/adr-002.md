# ADR-002: Homogeneous Condition Collections (No Mixing Uniform and Grid)

## Status

Accepted

## Date

2026-06-28

## Context

An instructor can add multiple conditions of the same type (e.g., two currents). Each condition can be either uniform (direction + speed) or grid-mapped. The question is whether these two formats can coexist in the same collection, requiring the system to evaluate and sum a uniform value with a grid interpolation.

## Decision

Condition collections must be homogeneous: all conditions in a collection are either all uniform or all grid-mapped. Mixing the two formats in the same collection is not permitted in this version. The architecture must be designed to allow this restriction to be lifted in a future version without data model changes.

## Alternatives Considered

### Alternative 1: Allow Mixed Collections

- **Description**: Uniform and grid conditions can coexist; the system evaluates each and sums the resulting vectors.
- **Pros**: Maximum flexibility for instructors.
- **Cons**: Significantly more complex UI (how does the instructor combine a slider with a file import in the same list?). Edge cases around how a uniform condition interacts with a grid that has zero values at certain cells.
- **Why rejected**: Adds UI and semantic complexity with unclear user benefit for the current scope.

### Alternative 2: Only One Format Per Simulation

- **Description**: A scenario can only have one format (uniform or grid) across all environmental types.
- **Pros**: Simplest data model.
- **Cons**: Too restrictive — an instructor might want a current map but only uniform wind, which is a valid and common scenario.
- **Why rejected**: Artificially constrains valid use cases.

## Consequences

### Positive

- Clear UI affordance: one condition type has one input paradigm.
- Simpler composition logic (uniform + uniform = vector sum; grid + grid = point-wise evaluation + sum).

### Negative

- Cannot combine a detailed current map with a quick uniform override in the same collection.

### Risks

- If the future requirement to mix formats arises, the data model may need a discriminated union — plan for this in the Type definition.
